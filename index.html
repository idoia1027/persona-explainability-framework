🏗️ Phase 1：游戏理解体系重构 ▶

设计理念：让算法真正“理解”游戏，不只是识别品类，而是拆解结构、机制、节奏、情感与体验。

逻辑结构：从原始标签出发，重构为多层语义体系（例）：
• 游戏内容理解-玩法机制 → 即时战斗 / 回合策略 / 模拟建造
• 游戏内容理解-题材世界观 → 奇幻 / 现代 / 科幻 / 历史
• 游戏内容理解-战斗模式 → 单人PVE / 非对称对抗 / 团队协作
• 游戏内容理解-节奏强度 → 高频操作 / 轻度休闲 / 阶段推进
• 游戏动机 → 成就驱动 / 探索驱动 / 社交驱动
• 付费动机 → 养成进阶 / 视觉收集 / 竞争优势

Showcase： Before： 《第五人格》仅有“非对称竞技、恐怖悬疑”。
After： 追逃对抗 × 情绪张力（高）× 心理博弈 + 团队配合；动机：成就 + 社交刺激。

成果意义：标签语义化与颗粒度提升，为后续画像和推荐提供可解释的语义底座。

语义层级化六维体系体验标签

🧬 Phase 2：用户兴趣映射与画像建模 ▶

设计理念：以标签体系为中介，将用户行为（游玩、付费、停留）结构化映射为兴趣标签权重。

逻辑结构：用户行为 → 标签映射 → TagShare 权重归一化 → 动机聚合 → 画像生成。

Showcase：系统会识别出某用户在《原神》《星穹铁道》中均有长期游玩，推导出其对“二次元幻想”“开放世界探索”“剧情沉浸”的高偏好权重，生成画像原型：剧情沉浸型玩家。

图示：用户画像样例（来源：用户理解点检 v2 系统）

🏷️ Phase 3：语义化人群体系构建 ▶

设计理念：让底层标签群“说人话”。通过语义归纳与自然语言表达，将复杂的标签组合转化为可识别的人设与社交标签。

逻辑结构：基于画像底层标签，通过规则归纳与语义分析生成人群原型，如“手感党”“策略脑”“剧情党”等。

图示：人群画像标签体系表样例（来源：画像原型表 v2）

💡 Phase 4：可解释性推荐与洞察落地 ▶

设计理念：通过语义化画像与游戏标签的结合，为推荐提供“理由”，让结果能被业务与用户读懂。

逻辑结构：画像特征 × 游戏标签 → 匹配动机 → 推荐理由生成 → 报告可视化。
